
Este proyecto es una página web sencilla hecha para el primer curso de programación.  
Incluye una página principal y varias secciones: Acerca de mí, Contacto, Red, Grado y un tema (Café).

El proyecto está organizado en carpetas y archivos para mantener una estructura clara y ordenada: la primera carpeta docs contiene todo el conjunto de las paginas y subpaginas que se han usado para esta web. la carpeta css contiene todos los estilos utilizados para cada pagina y la carpeta images contiene todas las imagenes usadas par las pagiunas.

Mis primeros problemas fueron con los estilos del CSS con que no adapatan los estilos a los htmls, para soluciones el problema use Copilot con el prompt "Tengo problemas con CSS de mi codigo para estilos, como envio la dirección a esa carpeta, dame un ejemplo".

Tmabien tuve problemas con los botones principales del pagina web, con que no tenian color o directamente no aparecían.
Para resolver este problema use el prompt "Tengo problemas con el codigo para hacer botones en visual studio code, no aparecen los botones superiores" con Microsoft 365 Copilot para resolver el problema.

Si ves el texto escrito en el codigo, el "header sticky" es un intento fallido de hacer que los botones esten fijos aun que se baje la pagina, no se ha cambiado porque no cambia ni destruye el codigo(no se tiene ningun problema con que este o no).
(ACTUALIZACION!): Al final funciono el header stinky para el movimiento de los botones superiores despues de terminar de hacer cambios con las demás subpaginas y el CSS., pero solo funciona en algunas paginas, no en todas.

Tuve un problema con el enlace entre todos las subpaginas y me daba el error 404, para reparar este error use el prompt "por alguna razon, al ser una sub pagina, si intento entrar a otra parte, me salta el error 404" con Chatgpt.

Este proyecto me permitió aplicar conocimientos básicos de HTML y CSS, organizar un sitio web con varias páginas interconectadas y resolver problemas reales.  
Aprendí sobre: manejo de rutas relativas y absolutas para enlazar CSS e imágenes, diseño responsive básico con contenedores y clases reutilizables, integración de imágenes y contenido temático (Café) y uso de Git y GitHub para control de versiones y publicación del proyecto.

En general, fue un trabajo entretenido y desafiante que cumplió los objetivos propuestos.
